# 格式标准详细规范

> 本文件为 [SKILL.md](../SKILL.md) 第一部分"格式标准"的详细参考。整理笔记时如遇格式疑问，查阅本文。

---

## 1. 标题层级体系详解

### 1.1 层级对应表

| Markdown 语法 | 级别 | 编号格式 | 使用频率 | 示例 |
|:-------------|:----:|:---------|:--------:|:-----|
| `#` | 一级 | 第X节 | 每节一个 | `# 第一节 国际私法的概念` |
| `##` | 二级 | 一、二、三… | 常用 | `## 一、国际私法的定义` |
| `###` | 三级 | （一）（二）… | 较常用 | `### （一）狭义说` |
| `####` | 四级 | 1.2.3.… | 极少用 | `#### 1. 代表人物` |

### 1.2 绝对禁止的写法

```markdown
# ❌ 第四章 国际私法概述        ← 禁止！一级不能是"章"
# ✅ 第一节 国际私法概述        ← 正确！一级是"节"

## ❌ 第一节 xxx               ← 禁止！"节"必须是一级(#)
## ✅ 一、xxx                  ← 正确！二级是"一、"
```

**关键原则**：输出文件中不出现"第X章"标题，因为每个输出文件本身就是一个小节。

### 1.3 本节评估标注

每节笔记的第一行（`# 标题`之后）必须附上评估：

```markdown
# 第一节 国际私法的概念

**本节评估**：教材约8页，知识点密度高，考试重要度高 → **A级**
```

评估维度：
- **教材约X页**：原始教材中该节的页数（估算即可）
- **知识点密度**：[高/中/低] — 信息密集程度
- **考试重要度**：[高/中/低] — 出题频率和分值权重
- **等级** → A/B/C 级
- **目标篇幅**：预计行数范围

## 2. 固定模块标签详解

### 2.1 标签使用规范

标签使用 `**加粗**` 格式，后跟冒号和内容：

```markdown
**核心概念**：涉外民事关系 = 含有涉外因素的民事法律关系。

**重要说明**：上述三个要素只要其一具有涉外性质，即为涉外民事关系。
```

### 2.2 全部标签速查

| 标签 | 适用场景 | 写法示例 |
|:-----|:---------|:---------|
| **核心概念** | 首次出现的术语定义 | **核心概念**：xxx |
| **重要说明** | 对前文的限定、补充或例外 | **重要说明**：但需注意… |
| **立法进步** | 新法对旧法的发展评价 | **立法进步**：新增了… |
| **学术争议** | 不同学说的对立观点 | **学术争议**：肯定说 vs 否定说 |
| **历史意义** | 学说/制度的历史地位 | **历史意义**：被誉为…之父 |
| **理论缺陷** | 某学说的内在问题 | **理论缺陷**：存在循环论证 |
| **方法论贡献** | 方法上的创新价值 | **方法论贡献**：首次采用…方法 |

### 2.3 使用原则

- 不是每段都需要标签，只在关键信息处使用
- 同一段落内不要堆砌超过 2 个标签
- 标签后的内容要简洁有力，不要大段复制原文

## 3. 表格格式详解

### 3.1 基本格式

```markdown
| 列标题1 | 列标题2 | 列标题3 |
|:-------:|:-------:|:-------:|
| 内容1   | 内容2   | 内容3   |
```

### 3.2 必须使用表格的场景

#### 场景A：学说对比（最常见）

```markdown
| 学说 | 代表人物 | 核心主张 | 评价 |
|:-----|:---------|:---------|:-----|
| 法则区别说 | 巴托鲁斯 | 区分"法则"的性质 | 近代国际私法萌芽 |
| 法律关系本座说 | 萨维尼 | 寻找法律关系的"本座" | 现代国际私法基础 |
| 既得权说 | 戴西 | 保护既得权利 | 英美法系影响深远 |
```

#### 场景B：构成要件 / 要素列举

```markdown
| 要素 | 内容 | 说明 |
|:-----|:-----|:-----|
| 主体 | 涉外 | 当事人或一方在域外 |
| 客体 | 涉外 | 标的物在域外 |
| 法律事实 | 涉外 | 产生/变更/消灭的事实发生在域外 |
```

#### 场景C：时间线 / 发展历程

```markdown
| 阶段 | 时间 | 特征 | 代表 |
|:-----|:-----|:-----|:-----|
| 萌芽期 | 古罗马-中世纪 | 属人法 vs 物法区分 | 法则区别说 |
| 近代 | 19世纪 | 法律本座说 | 萨维尼《现代罗马法体系》 |
| 现代 | 20世纪至今 | 最密切联系原则 | 美国《第二次冲突法重述》 |
```

#### 场景D：新旧法条对比

```markdown
| 对比项 | 旧法（《涉外民事关系法律适用法》实施前） | 新法（实施后） |
|:-----|:--------------------------------------:|:-------------:|
| 经常居所地 | 未规定 | 作为首要连结点 |
| … | … | … |
```

### 3.3 合并单元格（复杂结构场景）

**适用条件**：当内容的逻辑结构存在层级嵌套（如"大类 → 子类 → 具体项"），普通扁平表格无法清晰表达时，使用合并单元格格式。

**写法**：在 Markdown 表格中，将需要合并的单元格留空，视觉上与上方/左侧单元格形成"合并"效果。渲染为 HTML/PDF 时可通过样式实现真正的单元格合并。

**示例：多层级分类体系**

```markdown
**职业教育体系**：

| 类型 | 层次 | 实施主体 | 招生对象 | 学制 | 培养目标 |
|:-----|:-----|:---------|:---------|:-----|:---------|
| **职业学校教育** | 初等 | 初级职业中学 | 小学毕业生 | 3-4年 | 以文化课为重点，辅以职业教育和技能训练 |
| | 中等 | 职业高中、普通中等专业学校、技工学校、成人中等专业学校 | 初中毕业生 | 3年 | 中级技术人员、一般管理人员和技术工人 |
| | 高等 | 高等专科学校、职业大学、职业技术学院、技师学院 | 高中毕业生、中职毕业生 | 2-3年 | 应用型高级技术人员和管理人员 |
| **职业培训** | 从业前培训、转业培训、学徒培训、在岗培训、转岗培训等 | — | — | — | 以就业、转业、创业或提高职业技能水平为目的 |
```

**更多适用场景**：

| 场景类型 | 合并方式 | 示例 |
|:---------|:---------|:-----|
| 大类→子类嵌套 | 第一列大类名只写一次，下方留空 | 如上例"职业教育体系" |
| 制度构成的多层次展开 | 制度名称合并，子要素分行 | 某制度下多个主体×多个行为的矩阵 |
| 法条逐款解析 | 条文号合并，各款内容分行 | "第X条"占一列，各款分别一行 |
| 对比维度的交叉分析 | 行标题或列标题跨行/跨列 | 不同法域 × 不同制度的对比矩阵 |

**使用注意事项**：
- 留空合并的单元格在纯 Markdown 预览中可能显示不完美，但在 HTML/PDF 渲染后会正确合并
- 合并层级建议不超过 **3 层**，否则可读性下降
- 合并后仍需确保每行的信息完整，不能因为合并而丢失数据
- 第一列的"大类"建议用加粗标识，增强层次感

### 3.4 表格编写要点

- 表头要简短精准
- 单元格内文字尽量精炼（不是照搬原句）
- 对比类表格最后一列放"评价"或"特点"
- 超过 6 列的表格考虑拆分为两个表格
- 复杂层级结构优先考虑合并单元格，而非拆成多个小表

## 4. 强调与引用格式

### 4.1 加粗 (`**text**`)

**用于**：
- 核心概念术语首次出现：**法律规避**
- 学说名称：**意思自治原则**
- 重要关键词：**识别**、**反致**、**先决问题**

**不用加粗的**：
- 普通叙述文字（连续大段加粗 = 没有重点）

### 4.2 引用块 (`> text`)

**用于**：

**法典条文原文**：
> 《中华人民共和国涉外民事关系法律适用法》第2条："本法和其他法律对涉外民事关系法律适用没有规定的，可以适用与该涉外民事关系有最密切联系的法律。"

**经典学术表述**：
> 萨维尼指出："法律关系依其本座所归属的法律管辖。"（Savigny, *System des heutigen Römischen Rechts*）

**引用块规则**：
- 必须保留条文编号
- 长条文可截取关键句，但不可改写措辞
- 引用后可跟一句自己的概括性说明

### 4.3 分隔线 (`---`)

用于分隔同一节内不同二级标题下的内容块。不要过度使用——一般每个"一、"下面最多一条。

## 5. 英文翻译保留规则详解

### 5.1 必须保留翻译的类型

| 类型 | 处理方式 | 示例 |
|:-----|:---------|:-----|
| 法学概念（拉丁文/英文） | 括号内保留 | *jus gentium*（万民法）、*lex loci delicti**（侵权行为地法） |
| 学说名称（英文） | 括号内保留 | Statute Theory（法则区别说） |
| 人物姓名（原文拼写） | 括号内保留 | Bartolus（巴托鲁斯） |
| 法院判决名称 | 括号内保留 | *Babcock v. Jackson* |

### 5.2 可以省略翻译的类型

- 普通中文词汇（"方法"、"原则"、"条件"、"要素"等）
- 中国法律法规名称（无需英文）
- 纯描述性文字

### 5.3 格式要求

```markdown
萨维尼（Friedrich Carl von Savigny）在其著作《现代罗马法体系》
（*System des heutigen Römischen Rechts*）中提出法律关系本座说。
```

## 6. 其他格式细节

### 6.1 列表使用

- 有序列表（`1. 2. 3.`）：用于步骤、排名、有顺序的要点
- 无序列表（`-`）：用于并列要点、特征列举
- 嵌套列表不超过 3 层

**列表前置空行要求**：
- 当列表紧跟在段落、标签、引用块之后时，列表前必须有一个空行，否则 Python-Markdown 等解析器无法正确识别列表。

### 6.2 数字与日期

- 条文编号用中文：《民法典》第一百四十七条 或 《民法典》第147条
- 年份用阿拉伯数字：1998年、20世纪
- 章节编号统一用中文数字：第一章、第一节

### 6.3 特殊字符处理

- 书名号《》用于法规和书籍名
- 引号""用于直接引用
- 不使用斜体（Markdown 斜体在某些渲染器中显示不佳，用加粗替代）

### 6.4 输出净化规范

**不可见字符处理**：
- 输出前须确保不含零宽空格（U+200B）、零宽连接符（U+200D）、零宽非连接符（U+200C）、BOM（U+FEFF）、不间断空格（U+00A0）等不可见字符。
- 所有空格统一为普通半角空格（U+0020）。

**块级元素前置空行**：
- 任何表格（`|` 开头）与其上方非表格内容之间必须存在一个空行。
- 任何列表（`-` `+` `*` 或数字编号开头）与其上方段落、标签、引用块等非列表内容之间必须存在一个空行。
- 该规则适用于：普通段落、标签、列表、引用块、分隔线之后紧跟的表格或列表。
- 合并单元格的复杂表格同样遵守此规则。

**仅输出本小节正文内容**：
- 原文件可能包含部分非本节的内容，不得附带整理其他小节
- 除非确有必要，不得生成小节的总结
- 不要把质检清单、压缩率汇报写入笔记中
