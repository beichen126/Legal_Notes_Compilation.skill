# 压缩指南详细规范

> 本文件为 [SKILL.md](../SKILL.md) 第二部分"内容评估"和第三部分"详略取舍"的详细参考。整理笔记时如遇压缩程度疑问，查阅本文。

---

## 1. 等级划分操作指南

### 1.1 如何评定 A/B/C 级

**A级（80-100分）—— 详细展开，几乎不删**

判断依据：
- 考试高频考点（历年真题反复出现）
- 学科基础性概念（后续章节的基础）
- 包含重要法条原文或经典案例
- 学说争议密集（多家学说需要完整对比）

典型内容：
- 核心概念的完整定义和辨析
- 重要法律制度的构成要件与法律效果
- 代表性学说的详细论述
- 最高人民法院指导案例/公报案例
- 法律条文的新修或重大变化

**B级（50-79分）—— 平衡处理，适度压缩**

判断依据：
- 考试偶有涉及但非重点
- 属于知识体系的"填充"内容
- 以叙述性文字为主，概念密度中等

典型内容：
- 历史发展脉络（非核心制度）
- 次要学说的概述
- 一般性的分类或列举
- 制度的背景介绍

**C级（0-49分）—— 高度概括，只留骨架**

判断依据：
- 考试几乎不考
- 纯粹的背景性、介绍性内容
- 与其他章节高度重复的内容

典型内容：
- 冗长的立法背景介绍
- 国外制度的详细介绍（除非比较法考点）
- 教材中的扩展阅读类内容

### 1.2 等级与压缩率对照

| 等级 | 目标压缩率 | 笔记占原文比例 | 说明 |
|:----:|:---------:|:-------------:|:-----|
| A级 | 20-35% | 65-80% | 几乎不删，主要是语言精炼 |
| B级 | 35-55% | 45-65% | 删除冗余，保留框架 |
| C级 | 50-70% | 30-50% | 只留最核心要点 |

## 2. 各类型内容的处理方法详解

### 2.1 概念定义（必须详细保留）

**处理方式**：完整保留，可精炼措辞但不可删减要素。

```
原始文本（200字）：
    "涉外民事关系，又称为国际私法关系或国际民事关系，是指
     在主体、客体和权利义务事实这三个要素中，至少有一个要
     素与外国法域存在联系的法律关系。"

整理后（80字）：
    **核心概念**：涉外民事关系 = 主体、客体或法律事实中，
    至少一个要素具有涉外性质的民事法律关系。
```

**技巧**：用等号`=`或`即`替代冗长解释，删除"又称为"等同义替换。

### 2.2 学说主张（必须详细保留）

**处理方式**：

1. **保留三要素**：代表人物 + 核心主张 + 标志性表述
2. **对比用表格**
3. **评价精简为一句话**

```
原始文本（500字，描述某学说的来龙去脉）→
整理后（表格4行 × 3列，约150字）+ 一句评价
```

### 2.3 法条引用（必须原文保留）

**处理方式**：
- 完整条文 → 引用块（>）
- 长条文（超过100字）→ 引用关键条款 + 概括其余
- 必须包含：法规全称 + 条文编号 + 核心内容

```markdown
> 《中华人民共和国涉外民事关系法律适用法》第3条：
> "当事人依照法律规定可以明示选择涉外民事关系适用的法律。"

**重要说明**：该条确立了意思自治原则在涉外民事关系中的基础地位。
```

### 2.4 典型案例（保留关键要素）

**必须保留的四要素**：
1. **案名**（不能编造）
2. **基本案情**（2-3句话概括争议焦点）
3. **裁判要旨 / 判决结论**（法院的核心观点）
4. **意义 / 启示**（该案例的价值）

```markdown
**典型案例：奥汀案**（*Auten v. Auten*, 1954）

- **案情**：夫妻二人在英国结婚后移居美国，后一方在墨西哥取得离婚判决并再婚，另一方在纽约起诉要求确认婚姻有效。
- **争点**：离婚效力应适用何地法律？
- **裁判要旨**：法院提出"最重要联系"标准，综合考察多因素确定准据法。
- **意义**：标志着美国冲突法从僵硬规则向灵活方法的转型。
```

**省略什么**：
- 详细的诉讼程序经过
- 双方律师的完整辩论意见
- 下级法院的审理过程
- 非核心的事实细节

## 3. 压缩实操技巧

### 3.1 语言层压缩（不丢信息量）

| 技巧 | 原文 | 压缩后 |
|:-----|:-----|:-------|
| 用等号替代解释 | "所谓xxx，是指yyy" | **核心概念**：xxx = yyy |
| 删除过渡语 | "值得注意的是…"、"应当指出…" | 直接写要点 |
| 合并同类项 | 分三段说A、B、C的好处 | 一个表格列A/B/C |
| 用符号替代文字 | "第一…第二…第三…" | ①…②…③… |
| 删除自谦语 | "笔者认为"、"毋庸置疑" | 直接陈述观点 |

### 3.2 结构层压缩（重组不删信息）

| 技巧 | 适用场景 |
|:-----|:---------|
| 并列→表格 | 多个平行的特征、条件、要素 |
| 叙述→列表 | 描述性段落中的多个要点 |
| 长段→分段 | 一个大段落含多个子论点 |
| 对话式→陈述式 | "有人问…回答是…" → 直接给答案 |

### 3.3 信息层压缩（需谨慎，可能丢失信息）

⚠️ 以下操作必须在确保读者理解不受影响的前提下进行：

| 操作 | 安全前提 |
|:-----|:---------|
| 删除例证 | 该规则已有其他例子支撑 |
| 删除论证细节 | 结论本身已足够清晰 |
| 合并相似段落 | 两段确实说的是同一件事 |
| 概括引用 | 引用的核心意思已提取 |

## 4. 常见压缩错误

### ❌ 过度压缩的表现

1. **只留标题没有内容**：每个二级标题下只有一句话
2. **丢失论证逻辑**：直接给了结论但省略了推导过程
3. **法条变成自己的话**："大意是说…"而非原文引用
4. **案例只剩名字**："xx案"后面没有任何案情和要旨
5. **数字模糊化**："多个""若干""一些"替代了具体数字

### ✅ 正确压缩的表现

1. 读完后能复述出主要观点
2. 关键术语的定义完整准确
3. 法条能找到原文对应
4. 学说之间的区别清楚
5. 案例的启示和教训明确

## 5. 压缩率自查方法

### 5.1 快速估算

压缩率目测判断方法：
- 原始文件约 X 行 → 整理后约 Y 行
- Y/X ≈ 0.4~0.7 为理想范围

### 5.2 精确计算

```
压缩率 = (原文字符数 - 笔记字符数) / 原文字符数 × 100%
```

其中字符数 = 去除空格和换行后的纯文本字符。

### 5.3 如果压缩率异常

| 情况 | 可能原因 | 处理 |
|:-----|:---------|:-----|
| <20% | 几乎没改；可能是大段复制 | 加强语言精炼，删除冗余 |
| 20-35% | 内容较详细，A级正常 | 可接受 |
| 35-60% | 正常范围 | ✅ 通过 |
| 60-75% | 偏高但可能合理 | 检查是否丢了重要内容 |
| >75% | 过度压缩！ | ❌ 必须重做，补充内容 |
